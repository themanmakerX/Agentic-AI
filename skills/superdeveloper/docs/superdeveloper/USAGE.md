# USAGE.md Template

This is a reference template for projects that use `superdeveloper` as the orchestration layer.

## Purpose

Describe how an operator should use the system day to day.

## Suggested Sections

- What the system does
- When to use it
- How to start a task
- How platform detection works
- How the phase teams are expected to behave
- How to interpret the devil's advocate output
- What final artifacts are produced

## Example Content Pattern

- **Start**: explain the smallest command or prompt that begins the workflow.
- **Workflow**: describe the phase sequence in plain language.
- **Examples**: show one simple task and one complex task.
- **Troubleshooting**: list common failure modes and recovery steps.
- **Outputs**: explain where final docs and logs appear.

## Writing Rules

- Keep the language operational.
- Avoid architecture deep-dives.
- Prefer short commands, examples, and user-facing guidance.
- Keep platform-specific notes separate from the core usage path.
